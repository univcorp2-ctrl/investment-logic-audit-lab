# Research Plan: 歪み・泣き穴を探すための検証仮説

## 方針

派手な単発リターンより、以下を満たす候補だけを残します。

- バックテストだけでなくウォークフォワードでもプラス
- コストを上げても消えない
- ロングオンリーとロングショートの両方で説明できる
- 価格だけでなく、株式ではファンダメンタルの裏付けがある
- パラメータを少し変えても崩壊しない

## 候補仮説

### 1. 暗号資産の中期トレンド + ボラティリティ抑制

暗号資産は急騰急落が大きいため、常時保有よりトレンド期間だけ参加するほうが最大DDを下げられる可能性があります。

検証:

- BTC/ETH/SOLなどで63/126/252日モメンタム
- 取引コスト 5/10/25/50/100bps
- bull/bear/sideways regime別の成績

### 2. 株式のQuality + Value + Momentum合成

単純な低PERだけではバリュートラップを踏みやすいため、ROE、Debt/Equity、売上成長率、価格モメンタムを合わせます。

検証:

- ファンダメンタル上位を価格モメンタム上位で絞る
- 逆に、低品質・高レバレッジ銘柄のショート候補を作る
- 決算発表日・提出日ベースのラグを入れる

### 3. クロスアセットのリスクオン/オフ判定

株式指数・長期金利・ドル・BTCの組み合わせで、リスクオン期間だけ高ベータ資産を持つ仮説です。

検証:

- SPYが200日MA上、BTCが126日モメンタム上、VIXが低下基調など
- 条件が多すぎると過剰最適化しやすいため、3条件以内から開始

### 4. ありがちな落とし穴

- closeでシグナルを作って同じcloseで約定する
- 上場廃止銘柄を除いたデータだけで株式戦略を作る
- 暗号資産の取引所手数料・スリッページを過小評価する
- 最高Sharpeの単一パラメータだけを採用する
- ファンダメンタルデータの公表日前の値を使う

## 採用基準

- OOS Sharpe > 0.7 を目安。ただし低ボラ戦略はSortinoや最大DDも重視
- 最大DDがBuy & Holdより明確に低い
- fee 25bps以上でも総リターンが大きく崩れない
- train上位パラメータがtestでも極端に順位反転しない
- 月次・四半期別に一部期間だけでなく複数regimeで残る
